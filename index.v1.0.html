LPIC-1: System Administrator

Overview

  • LPIC-1 is the first certification in LPI’s multi-level Linux professional certification program.
  • LPIC-1 validates the candidate's ability to perform maintenance tasks in real world system administration.

Official information

  • Current Version: 5.0 (codes 101-500 & 102-500).
  • Objectives: 101-500, 102-500.
  • Prerequisites: No.
  • Requirements: Passing exams 101 and 102.
  • Validity Period: 5 years.

Goals & requirements

  • Understand the architecture of a Linux system.
  • Install and maintain a Linux workstation.
    Including X11 and setup it up as a network client.
  • Work at the Linux command line.
    Including common GNU and Unix commands.
  • Handle files and access permissions as well as system security.
  • Perform easy maintenance tasks.
    Help users, Add users to system, Backup and restore and Shutdown and reboot.

Exam 101

Topic 101: System architecture

Configure hardware settings

Overview

  • https://developer.ibm.com/tutorials/l-lpic1-101-1/
  • Learn to configure and check the hardware for your Linux system.
  • Prepare for Objective 101.1 in Topic 101 of the Linux Server Professional (LPIC-1) exam 101.

Hardware settings

  • Settings can be configured via either BIOS (Basic Input-Output Services) or UEFI (Universal Extensible Firmware Interface) firmware.
  • The settings can be accessed when turn on computer then press a special button or key sequence.
  • Common settings:
    • Date and time.
    • Peripherals.
    • Boot order.

Mass storage devices - Hard drives

  • Hard drives come in a sealed unit and contain several recording surfaces or platters.
  • There are many standard interfaces between the controller card and the computer.

Mass storage devices - CD and DVD drives

  • CD, DVD and Blu-Ray drives use optical storage to hold data.
  • They use the same kinds of interfaces used for hard drives.

Mass storage devices - Flash or thumb drives

  • These drives use nonvolatile RAM to store data.
  • They are reliable, fast, high-capacity and can be reused many times.

Mass storage devices - Solid-state drives

  • A newer form of drive .
  • They are light, fast, use less power and do not suffer from mechanical failure (?).

Hot plug and cold plug

  • The term hot plugging refers to devices that are removable from the computer every time.
  • The term cold plugging refers to devices that are physically attached to the computer at boot time.

Linux

  • Linux is an operating system to a computer’s basic functionality. It forms the interface between the hardware and applications as well as the interface between the hardware and users.
  • Linux kernel performs the actual operating system tasks of Linux. It takes care of elementary functions like memory and process management and hardware control.

Virtual files for hardware and system information

  • Virtual filesystems provide an in-memory look at your system as the kernel sees it.
  • The virtual filesystems provide insight into kernel data structures and into your system hardware.

Sysfs virtual filesystem

  • The sysfs contains information about kernel objects to processes running in user space.
  • The sysfs is an in-memory filesystem that is mounted at /sys.

Procfs virtual filesystem

  • The procfs provides insight into many of the kernel data structures and even allows a few to be changed on the fly.
  • The procfs presents process information and kernel processes.
  • The procfs is an in-memory filesystem that is mounted at /proc.

The lsdev command

  • The procinfo package contains lsdev command.
  • The lsdev command presents the information from /proc by gathering the DMA, IRQ, and I/O port information for each device into a simple tabular format.

udev and /dev

  • udev is a device manager needed for hot plugging devices.
  • Information about configured and active devices is contained in the /dev virtual filesystem.

Tools and utilities

  • Use lspci command without options to see a basic listing of your PCI devices.
  • Use lsusb command to display information about your USB devices.

Message passing and D-Bus

  • D-Bus is a message bus system, a simple way for applications to talk to one another.
  • D-Bus supplies both a system daemon and a per-user-login-session daemon.

Kernel modules

  • Kernel modules are pieces of code that can be loaded and unloaded into the kernel upon demand.
  • Kernel modules extend the functionality of the kernel without the need to reboot the system.
  • Kernel modules are located in /lib/modules and have had the extension .ko ("kernel object") since version 2.6.
  • The lsmod command formats the information from /proc/modules to give you a current status of the modules in your Linux system.
  • The modinfo command gives you information about a module.
  • The modprobe command can each insert or remove one module from the system.
